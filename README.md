# 云游东洋预约系统 - 讽刺性互动体验项目

##
预览页面
https://dongyang.01101.top
##
项目简介

这是一个讽刺性的虚拟景区预约系统，旨在通过夸张的用户体验来揭示和批评当前中国景区预约系统中存在的各种问题。项目采用现代化的古风美学设计，对比糟糕的后端体验，深刻反映了"前端精美，后端垃圾"的现实状况。

## 🎭 项目特色

### 视觉设计
- **古风美学**：采用传统中华美学设计语言，金色、红色、深青色的经典配色
- **现代技术**：响应式设计、流畅动画、精美交互效果
- **视觉对比**：精美的前端界面与糟糕的用户体验形成强烈反差

### 功能特色
- **多重人机验证**：图片验证码、滑动验证、数学验证码随机出现
- **虚假排队系统**：2847人排队的假象，实际无人排队
- **系统卡顿模拟**：真实的响应延迟和按钮变灰体验
- **周一闭馆限制**：模拟真实景区的运营限制
- **强制协议阅读**：必须滚动到95%才能继续的冗长协议
- **抽签预约机制**：28947人抢50个名额的绝望体验
- **弱智AI客服**：只会按部就班回复的无用客服系统
- **虚假库存显示**：有票显示无票，无票显示有票的恶心体验

## 🎯 体验流程

### 第一阶段：希望
1. **精美加载动画**：4秒古风加载界面，营造专业感
2. **现代化首页**：精美的英雄区域和特色功能展示
3. **智慧预约承诺**：99.9%可靠性、24/7服务等虚假宣传

### 第二阶段：困扰
4. **登录验证地狱**：多重人机验证，验证码通知弹窗
5. **强制协议阅读**：冗长的用户协议必须完整滚动
6. **虚假排队折磨**：2847人排队，随机异常和超时

### 第三阶段：挫败
7. **周一闭馆检查**：选择周一自动拒绝
8. **抽签系统绝望**：28947人抢50个名额，95%失败率
9. **系统卡顿体验**：按钮变灰、响应缓慢、强制等待

### 第四阶段：愤怒
10. **虚假库存信息**：明明有票却显示售罄
11. **弱智客服折磨**：AI客服只会复制粘贴标准答案
12. **支付系统崩溃**：最后一步的各种支付失败

### 第五阶段：觉醒
13. **通关页面揭露**：显示真实余票数量和系统现状
14. **现实分析报告**：深度剖析当前预约系统问题
15. **改进建议提供**：给出具体的解决方案

## 🔧 技术实现

### 前端技术栈
- **HTML5**：语义化标签，无障碍访问支持
- **CSS3**：Flexbox/Grid布局，CSS动画，响应式设计
- **JavaScript ES6+**：模块化开发，异步处理，DOM操作
- **字体资源**：Noto Serif SC（宋体）、Ma Shan Zheng（书法体）
- **图标库**：Font Awesome 6.0

### 核心功能模块
- **script.js**：主要业务逻辑和用户交互
- **enhanced-features.js**：增强功能（协议阅读、抽签系统）
- **compact-ui.js**：界面紧凑化优化
- **ai-service.js**：AI客服系统

### 样式模块
- **styles.css**：主要样式和古风设计
- **captcha-styles.css**：人机验证相关样式
- **compact-ticket-styles.css**：紧凑票种选择样式
- **ai-service-styles.css**：AI客服界面样式
- **payment-styles.css**：支付相关样式
